# RepoGenius AI

**AI-Powered Multi-Agent Repository Optimization Platform**

Submit a GitHub repository URL → Get a complete optimization report with prioritized recommendations, architecture diagrams, security audit, and a sprint-based improvement roadmap — all generated by parallel AI agents using free open-source tools.

---

## What It Does

1. You enter a **GitHub repo URL** + credentials in the web UI
2. The platform **clones** the repository
3. **8 AI agents run in parallel** analyzing security, architecture, code quality, dependencies, technical debt, and more
4. You get an **Optimization Report** with:
   - **Optimization Score** (0–100)
   - **Quick Wins** — top 10 things to fix right now
   - **Sprint Roadmap** — 4-sprint improvement plan
   - **Architecture Diagrams** (auto-generated Mermaid)
   - **Security Findings** (with OWASP + CVE references)
   - **Technical Debt** estimate (hours + $USD)
   - **Engineering Maturity** level (Beginner → Enterprise)

**The platform NEVER modifies your code.** It only provides recommendations.

## Quick Start

```bash

git clone https://github.com/your-username/githubrepooptimizer.git
cd githubrepooptimizer

# 2. Copy environment variables
cp .env.example .env
# Edit .env with your settings (see Configuration below)

# 3. Start everything with Docker
docker compose up

# 4. Open the UI
# Frontend: http://localhost:3000
# API Docs: http://localhost:8000/api/v1/docs
# Health:   http://localhost:8000/health
```

That's it. Submit a GitHub repo URL in the UI and watch the agents work.

## Agents (All Run In Parallel)

| Agent | What It Does | Free Tools Used |
|-------|-------------|-----------------|
| **Security** | Finds secrets, CVEs, insecure patterns | detect-secrets, bandit, semgrep, osv-scanner, trivy |
| **Code Quality** | Measures complexity, duplication, dead code | radon, flake8, vulture, jscpd, pylint |
| **Architecture** | Detects patterns/anti-patterns, generates diagrams | tree-sitter, madge, pydeps, Mermaid |
| **Dependency** | Audits outdated/vulnerable packages + licenses | pip-audit, osv-scanner, npm audit, license-checker |
| **Technical Debt** | Estimates debt in hours + $, prioritizes fixes | radon, pylint, jscpd, coverage.py |
| **Executive CTO** | Executive summary + strategic recommendations | Ollama (free local LLM) |
| **Optimization** | Merges findings, generates roadmap + score | Ollama (free local LLM) |
| **Repo Understanding** | Extracts metadata — languages, frameworks, contributors | GitPython, tree-sitter |

**All 8 agents run simultaneously** using `asyncio.gather`. Total analysis time: ~60-90 seconds.

## Key Design Decisions

1. **All agents parallel** — `asyncio.gather` dispatches all 8 agents simultaneously. Analysis takes ~60s, not 8×60s.

2. **Free by default** — Ollama runs locally at zero cost. No API keys required for basic analysis. Paid providers (Claude, Bedrock) are optional fallback for higher quality.

3. **Parse once, share everywhere** — `SharedAnalysisContext` parses the repo ONCE (AST, dependencies, symbols). Agents never re-read files independently. Saves 80% of processing time.

4. **Never modifies code** — The platform produces recommendations and reports. It never pushes commits, creates PRs, or changes any file in your repository.

5. **Plugin architecture** — Add a new agent by dropping a Python file in `agents/core/`. No orchestrator changes needed.
